# PetCare

PetCare — это mobile-first приложение для владельцев домашних животных, которое объединяет цифровой профиль питомца, документы, ИИ-помощника, совместный доступ и карту ветклиник и dog-friendly мест в одном продукте.

Проект ориентирован в первую очередь на владельцев собак и кошек и развивается как `web + Android` решение через `PWA`, с дальнейшей стратегией публикации в `Google Play` и `RuStore`.

## Быстрый старт


Скопируйте env файл
```bash
cp .env.example .env
```

Перейдите в директорию проекта
```bash
cd petcare
```

Запустите контейнеры
```bash
make build
```
### Сервисы будут доступны по следующим адресам:     
frontend: http://172.18.0.5:5173/  
backend: http://0.0.0.0:8000 

При завершении работы остановите все сервисы
```bash
make down
```

Другие готовые команды можно найти в Makefile.

## Идея проекта
PetCare решает проблему фрагментированного ухода за питомцем.

Сегодня важная информация о животном обычно разбросана по разным источникам:
- документы лежат в галерее, чатах и заметках;
- советы по уходу приходится искать в случайных статьях и сообществах;
- ветеринаров и dog-friendly места сложно подбирать с учетом особенностей конкретного питомца;
- в поездке, после переезда или в непредвиденной ситуации пользователь быстро теряет привычный контекст.

Особенно остро эта проблема ощущается:
- у новичков, которые только начинают ухаживать за первым питомцем;
- у владельцев, столкнувшихся с нестандартной ситуацией;
- у пользователей, которым нужен постоянный мобильный доступ к данным питомца.

## Что умеет приложение
В целевой модели пользователь может:
- создать профиль питомца с основными характеристиками;
- хранить фото и документы питомца;
- получать советы от ИИ-помощника;
- искать ветклиники и dog-friendly места;
- делиться карточкой питомца с другим человеком;
- пользоваться приложением как PWA на Android;
- иметь ограниченный офлайн-доступ к самым важным данным питомца.

Ключевые продуктовые решения, уже зафиксированные в документации:
- способы входа: `email/password + Telegram`;
- Telegram используется и как отдельный канал, и как альтернативный способ входа в приложение;
- первая целевая платформа: `Android + web`;
- `iOS` не является обязательным приоритетом первой версии;
- ИИ-чат должен быть привязан к конкретному питомцу;
- контекст профиля питомца в ИИ-чате подключается только по явному выбору пользователя;
- `shared user` должен иметь почти все сценарии по питомцу, кроме редактирования данных питомца и управления шарингом;
- работа с CSV и картографическими данными должна быть доступна только администраторам.

## Целевая аудитория
Основные пользователи PetCare:
- новички, недавно заведшие первого питомца;
- городские владельцы собак и кошек;
- семьи или несколько людей, совместно ухаживающие за питомцем;
- пользователи, которым важен быстрый доступ к данным питомца в дороге, поездке или после переезда.

## Текущее состояние проекта
Сейчас проект уже представляет собой рабочий MVP с реализованными frontend- и backend-частями.

### Что уже реализовано
- регистрация и вход по `email/password`;
- восстановление сессии через `refresh` flow;
- CRUD профилей питомцев;
- загрузка и обновление фото питомца;
- загрузка, просмотр, скачивание и удаление документов;
- совместный доступ к питомцу через инвайты;
- карта с ветклиниками, dog-friendly местами и grooming-салонами;
- ИИ-чат;
- профиль пользователя;
- PWA-конфигурация и production build frontend;
- Telegram bot как отдельный канал продукта;
- CSV-импорт картографических данных на backend;
- единый формат ошибок API.

### Что ещё не доведено до целевой модели
- offline-сценарии ограничены и не покрывают весь продуктовый объём;
- push-уведомления не реализованы;
- часть продуктовых сценариев ещё не доведена до финального UX;
- Telegram login поддержан на backend, но не доведён до полноценного пользовательского сценария во frontend UI.

## Архитектура проекта

### Backend
Backend построен как отдельный асинхронный API-сервис на:
- `FastAPI`
- `SQLAlchemy asyncio`
- `PostgreSQL`
- `Alembic`
- `MinIO`
- `OpenRouter`
- `aiogram`

Ключевые backend-домены:
- `auth`
- `users`
- `pets`
- `pet photos`
- `pet documents`
- `sharing / invites`
- `map points`
- `llm chat`

### Frontend
Frontend уже реализован как отдельное SPA-приложение на:
- `React`
- `TypeScript`
- `Vite`
- `TanStack Router`
- `TanStack Query`
- `React Hook Form`
- `Zod`
- `Tailwind CSS`
- `vite-plugin-pwa`

На текущем этапе управление состоянием строится в основном на:
- `React hooks` для локального UI-state;
- `TanStack Query` для server state и кэша API;
- `TanStack Router` для route state;
- отдельном auth-session модуле для access token и статуса сессии.

Frontend работает как клиент к backend API через `/api` proxy в dev и same-origin схему в production.

## Что особенно важно в архитектуре

- frontend и backend связаны через единый `/api`-слой;
- авторизация построена на связке `access token + refresh cookie`;
- загрузка фото и документов реализована через `presigned URL`;
- данные UI синхронизируются через `TanStack Query`;
- backend разделён на `api / service / repository / model` слои.
